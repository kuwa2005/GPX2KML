# セキュリティ

## スコープ

- 対象: ブラウザ上で動作する単一 HTML（`GPX2KML2.html`）。サーバへの GPX アップロードは行いません。
- 最終レビュー（記録）: 2026-04-02

## 脅威と対策（実装状況）

| 項目 | 内容 |
|------|------|
| KML / XML インジェクション | ドキュメント名・プラスマーク名・ウェイポイント名など、ユーザー由来のテキストは `escapeXml` でエスケープしてから KML に埋め込みます。 |
| 座標・色の異常値 | 座標は `fmtKmlCoord` で有限数のみ出力。KML の `<color>` は 8 桁の 16 進（AABBGGRR）に `sanitizeKmlColor8` で正規化し、それ以外は既定色にフォールバックします。 |
| DOM ベース XSS | `innerHTML` や `eval` によるユーザー入力の挿入は行っていません。プレビューは `textarea` の `value` で表示しています。 |
| テーマ Cookie | `gpx2kml_theme` は `light` / `dark` のみコード側で設定。読み取りは表示切替のみに使用します。`SameSite=Lax` を付与しています。 |
| プライバシー | GPX の処理はクライアント内完結。外部 API へトラックを送信しません。 |

## 既知の制約・残存リスク

1. **サプライチェーン（CDN）**  
   Tailwind（開発用 CDN）、JSZip、Google Fonts を外部から読み込みます。改ざん耐性を高めるには Subresource Integrity（SRI）の付与や、ファイルのローカルバンドル化が有効です。

2. **生成 KML 内の外部 URL**  
   ポイント用アイコンとして Google ホストの PNG を参照します。閲覧環境によってはネットワーク負荷や可用性の影響を受けます（表示用アセットであり、アプリ本体の XSS 直結ではありません）。

3. **リソース枯渇（DoS 的利用）**  
   巨大な GPX や「トラック端マーカー：全て表示」による大量 Placemark で、ブラウザのメモリ・応答が悪化する可能性があります。信頼できないファイルは控えめな設定で開いてください。
